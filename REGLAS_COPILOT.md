# Reglas de trabajo con GitHub Copilot

1. **Cambios quirúrgicos** — Todos los cambios tienen que ser lo más precisos y acotados posible, tocando únicamente lo necesario. No modificar nada que no esté relacionado con el pedido, a menos que el usuario indique explícitamente lo contrario.

2. **Soluciones simples** — Siempre optar por la solución más sencilla y directa. No agregar complejidad innecesaria, no refactorizar de más, no "mejorar" cosas que no se pidió mejorar.

3. **Sintaxis correcta** — Todo el código generado o modificado debe tener una sintaxis válida y coherente con el lenguaje y framework del proyecto (React Native / Expo / TypeScript).

4. **No inventar información** — Si se necesita conocimiento sobre tablas de base de datos, endpoints de API, estructuras de datos, configuraciones externas o cualquier otra información que no esté visible en el contexto, **preguntar siempre al usuario**. Nunca asumir, nunca inventar.
